2. I18n Template Translation

i18n template translation

The i18n template translation process has four phases:

  1. Add i18n attributes to your components HTML.
  2. An angular i18n tool extracts the marked messages into an industry standard translation source file.
  3. A translator edits that file, translating the extracted text messages into the target language, and returns the file to you.
  4. The Angular compiler imports the completed translation files, replaces the original messages with translated text.

I18n workflow

i18n templates  -------->  translation  --------> merge translations

Mark text with the i18n attribute

  1. It can easily mark DOM elements that need to be internationalized by adding the i18n directive (as an attribute).
  2. The i18n attribute can be used without a value, or you can add a description and a meaning as value (separated by a pipe)
  3. The parser wont translate your HTML tags, but will replace them by placeholders in the translation dictionary
  4. It is possible to translate element attributes as well, by adding i18n-[the attribute] to your element, you can mark a whole group of elements to be translated by wrapping them in a i18n-comment or using the ng-container construct and there is also support for pluralization.

Have a look at some of the syntax:

<span i18n>Hello i18n!</span>

<span i18n="User welcome|An introduction header for this sample">Hello i18n!</span>
<span i18n="other description|this is a machine">Hello Angular</span>

<!-- i18n -->
<div>I don't output any element either</div>
<!--/i18n-->

<ng-container i18n>
  I don't output any element
</ng-container>

Handle singular and plural

  1. Different languages have different pluralization rules.
  2. Angular uses the standardized ICU Message Format that derives from the Common Locale Data Repository (CLDR).
  3. Suppose your application says something about a collection of apples. In English, depending upon the number of apples, you could display "no apples", "one apple", "two apples", or "a apple pack". Other languages might express the cardinality differently.
  • The first parameter is the key. It is bound to the component property (apples) that determines the number of apples.
  • The second parameter identifies this as a plural translation type.
  • The third parameter defines a pluralization pattern consisting of pluralization categories and their matching values.
  1. You can handle this with a select translation.You choose among alternative translation based on a string value instead of a number.
  2. The following format message in the component template binds to the component's friendGender property, which outputs either an "male", "female" or an "other".

Have a look at some of the syntax:

<button i18n>
  {wolves, plural, 
  =0 {no wolves}
  =1 {one wolf}
  =2 {two wolves}
  other {a wolf pack}}
</button>

<button i18n>
  { friendGender, select,
    male { Invite him }
    female { Invite her }
    other { Invite them }
  }
</button>
